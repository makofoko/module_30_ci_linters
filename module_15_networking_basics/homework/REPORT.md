# REPORT.md

## Уровень зрелости API
Изначальный контракт соответствует уровню 1–2 модели зрелости Ричардсона: действия отражены в названиях (`add-room`, `get-room`), отсутствует версионирование и часть стандартных методов.

## Внесённые улучшения
- Добавлено версионирование: `/api/v1`.
- Эндпоинты приведены к ресурсному стилю: `rooms`, `bookings`.
- Введены стандартные методы: `GET`, `POST`, `PUT`, `DELETE`.
- Фильтры перенесены в query‑параметры (`checkIn`, `checkOut`, `guestsNum`).
- Унифицированы ответы:
  - Успех: возвращается созданный ресурс и статус `201 Created`.
  - Ошибка: формат `{ "error": { "code": "...", "message": "..." } }`.
- Уточнены названия: множественное число для коллекций, `roomId`, `bookingId`.

## Обоснование
- Ресурсные пути и методы делают API предсказуемым и совместимым с клиентами.
- Версионирование предотвращает ломку контрактов при будущих изменениях.
- Унификация ошибок упрощает обработку на клиенте.
- Query‑параметры для фильтрации — стандартная практика HTTP.

## Postman‑коллекция
Коллекция `hotel-api-v1.postman_collection.json` содержит:
- Rooms: `POST /rooms`, `GET /rooms`, `GET /rooms?filters`, `PUT /rooms/{id}`, `DELETE /rooms/{id}`.
- Bookings: `POST /bookings`, `GET /bookings`, `DELETE /bookings/{id}`.
Все запросы снабжены тестами и используют переменные окружения.
