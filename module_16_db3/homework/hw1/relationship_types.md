# Классификация связей между таблицами

## 1. Один ко многим (One-to-Many)
- **director → movie_direction**
  - Один режиссёр может снимать несколько фильмов.
  - `director.dir_id` → `movie_direction.dir_id`

- **movie → oscar_awarded**
  - Один фильм может получить несколько наград.
  - `movie.mov_id` → `oscar_awarded.mov_id`

## 2. Многие ко многим (Many-to-Many)
- **movie ↔ director**
  - Через `movie_direction`: один фильм может иметь нескольких режиссёров, один режиссёр — несколько фильмов.

- **movie ↔ actors**
  - Через `movie_cast`: один фильм может иметь много актёров, один актёр может играть в нескольких фильмах.

## 3. Один к одному (One-to-One)
- В данной схеме **не реализовано явно**, но можно было бы добавить, например, `movie → movie_poster`, если у каждого фильма только один постер.

## 4. Отношение с атрибутом
- **movie_cast** содержит дополнительный атрибут `role`, описывающий роль актёра в фильме.

## 5. Составной ключ
- В `movie_cast` и `movie_direction` используется составной первичный ключ, чтобы избежать дублирования связей.
